## Design Context

### Users

Developers and technical PMs who build AI-assisted HTML presentation workflows. Comfortable with JSON and prompt engineering. Work iteratively: upload an HTML slide template → assign content zones via a visual DOM tree panel or CodeMirror editor → generate a structured AI recipe prompt → paste the AI's JSON response → apply and preview. Desk-based focused sessions, often in low-light environments. This is a professional instrument, not a consumer product.

**Job to be done**: Eliminate the manual work of populating HTML slide decks with AI-generated content. Users want a reliable, repeatable pipeline. Correctness and workflow efficiency matter far more than visual delight.

### Brand Personality

**Three words**: Precise. Purposeful. Understated.

The interface should feel like a well-engineered instrument — the kind of tool a developer trusts because it gets out of the way. No marketing energy. No celebration. Just clarity and control.

**Emotional goal**: Confidence. Users navigate a complex multi-step pipeline (upload → zone assignment → recipe generation → AI JSON → apply → preview). At every step they should feel oriented and in control, never second-guessing what the tool is doing or what comes next.

### Aesthetic Direction

**Theme**: Dark. Always. Users work in focused sessions, often at night or in low-light environments. The existing dark teal palette (`--bg-primary: #061210`, `--accent-primary: #4CAF80`) avoids the clichéd cyan/purple AI aesthetic and uses a nature-derived green that feels calm and deliberate rather than electric. Keep it.

**Visual tone**: Technical-minimal. Tight, information-dense layouts. The core UI surfaces are a DOM tree panel, a CodeMirror editor, a recipe textarea, and a scaled iframe preview — inherently dense. Whitespace is purposeful and hierarchical, not decorative.

**References**: Vercel dashboard, Linear, Warp terminal. Clean, dark, information-forward. Hierarchy through weight and spacing, not decoration or color splashes.

**Anti-references**: SaaS marketing sites. No hero sections. No gradient text. No floating card grids. No outer glows as primary hover affordances. No bounce animations. No sparklines used decoratively.

### Typography

**Note on Inter**: The CLI detector flagged Inter as overused (`[overused-font]`). However, given the existing codebase and the tool's density, replacing Inter would be disruptive without clear gain. If typography is revisited, consider **Geist** (Vercel's own font — clean, slightly narrower than Inter, engineered-tool feel) or **Figtree** as alternatives that are not on the reflex list. Either way, the font must actually be loaded — currently neither Inter nor JetBrains Mono is fetched from anywhere.

**Monospace**: JetBrains Mono for all code/data surfaces (CodeMirror editor, recipe textarea, JSON input, debug modal preview). Must be loaded.

**Rule**: Monospace for data, proportional for UI chrome. Never mix roles.

### Design Principles

1. **Hierarchy through density, not decoration.** The interface carries a lot of simultaneous information (DOM tree, zone badges, slide tabs, recipe panel, preview iframe). Create clarity through typographic contrast and spatial rhythm — not borders, icons, or color splashes.

2. **Every element earns its place.** The design is already lean. New additions must justify themselves functionally. If something can be removed, it should be.

3. **Dark mode is the correct mode.** The dark teal palette is the right call for this tool and this user. No light-mode variants unless explicitly requested.

4. **Monospace for data, proportional for UI.** Recipe/JSON/editor areas use JetBrains Mono. All UI chrome uses the proportional font. Do not mix roles.

5. **Motion is information, not decoration.** Step-transition animations (`slideInRight`/`slideInLeft`) communicate navigation direction. Any new motion must tell the user something. All motion must respect `prefers-reduced-motion`.

6. **Keyboard navigation is non-negotiable.** This is a developer tool. Every interactive element must have a visible `:focus-visible` ring. WCAG AA is the floor.
