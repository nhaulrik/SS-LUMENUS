Use Case 1

Managing Templates Centrally

Setting

An analysis phase has started in a project. Numerous workshops are being scheduled across multiple teams within the project.

Problem

Management wants all workshops to use the same format. The current approach — a shared PowerPoint template — becomes a distribution nightmare whenever updates are needed. Who has the latest version?

Solution with SOLON

The management group creates a project in SOLON Slide Studio hosted in a shared Git repository. All teams generate their workshop presentations from the same centrally-managed templates. When management pushes a template update to Git, every team automatically uses the new version on their next generation. Teams can also branch templates for custom needs — and propose changes via Pull Request for management review.

2
Use Case 2

Adding Fields Without the Grind

Setting

A team has prepared a series of customer workshops. A PowerPoint with 70 slides — one per requirement — is ready to go.

Problem

A decision is made: add a "Total Estimate" field to every slide. A team member is delegated to manually update all 70 slides by looking up values in Jira one by one. Hours of tedious, error-prone work.

Solution with SOLON

The team member adds the new field to the template, ensures the estimate data is present in the AI context, and describes how the AI should populate it. Then hits Generate. The entire updated presentation is ready in minutes — not hours — with accurate data pulled directly from the source.
